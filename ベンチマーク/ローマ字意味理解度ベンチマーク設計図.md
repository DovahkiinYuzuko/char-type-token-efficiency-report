# ローマ字意味理解度ベンチマーク設計図
RakutenAI 3.0によるまとめ

## この文書について

この文書は「ローマ字意味理解度ベンチマーク構想.md」で述べた問題意識をもとに、
ベンチマークの設計要素を整理するための草案である。

現時点では個人レベルで検討している段階であり、
Issue化や大規模運用は前提としていない。
まずはベンチの基礎設計と、個人で進められる最小構成を明確にすることを目的とする。

## 構想メモ

### ベンチの目的
- ローマ字で書かれた日本語入力に対して、LLMがどれだけ正確に意味復元できるかを測る
- 特に、現実の雑な対AI入力に対する頑健性を評価する
- 単なる文字列一致ではなく、語境界・助詞・否定・条件・固有名詞・文体の保持も見る

### 基本方針
- まず正規日本語原文を作る
- 原文に属性ラベルを付ける
- そこから条件別にローマ字派生文を生成する
- モデルに「要約せず自然な日本語へ復元」させる
- 原文と比較して採点する

### 主タスク
#### 正規日本語復元
- 与えられたローマ字文を、要約や言い換えをせず、自然な日本語に復元できるかを見る
- これをベンチの基礎タスクにする

#### 補助タスク
- 内容QA
- 意図分類
- 文体保持復元
- ただし初期段階では復元タスクを優先する

### データ属性軸
#### 意味条件
- 単純叙述
- 否定
- 比較
- 条件文
- 依頼
- 感情表現
- 曖昧参照

#### 文体条件
- 標準
- 会話
- 掲示板
- SNS
- 技術
- コピペ調

#### 語彙条件
- 一般語彙
- スラング
- ネットミーム
- 専門用語
- 固有名詞混在
- 英語混在

#### 区切り条件
- 空白あり
- 空白なし
- 文節ごと空白
- 句読点あり
- 句読点なし

#### ローマ字変換規則
- Hepburn
- Kunrei
- Mixed
- Casual
- IME-like

#### ノイズ条件
- typoなし
- 軽微 typo
- 重複子音
- 文字脱落
- 文字重複
- その他の崩れ

### セット構成
#### Core Set
- 一般的な日本語
- 標準文・会話文中心
- 依頼、否定、条件文など
- ベンチの主軸

#### Internet Style Set
- 掲示板語
- ネットスラング
- 崩れた口語
- 感情過多文体
- 現実の雑入力への頑健性を見る

#### Meme Set
- 吉野家コピペ
- 許してクレメンス
- 〇〇ンゴねぇ
- その他定型ミーム
- 文化依存・長文保持・文体保持を見る拡張セット

### 採点基準
#### Level 1: 完全一致
- 原文と完全に一致するか

#### Level 2: 正規化一致
- 句読点差
- 軽微な表記揺れ
- 一部の漢字かな差
- これらを無視して一致するか

#### Level 3: 意味一致
- 意味が保たれているか
- 要約していないか
- 否定・条件・比較が壊れていないか
- 固有名詞や数値が落ちていないか

### 誤答分類
- 語境界誤り
- 助詞誤り
- 否定反転
- 条件誤解
- 固有名詞誤認
- 英語混在誤解
- 要約化
- 文体消失
- ミーム正規化しすぎ

### 設計上の注意
- 2chコピペは強い素材だが、ベンチ全体の中心にはしない
- 2chコピペは Internet Style Set や Meme Set の強力な評価素材として使う
- `shi/si` などは typo ではなく、ローマ字変換規則の差として扱う
- 文体と語彙は分ける
- 完全一致だけで採点しない
- 最初から全部盛りにしない
